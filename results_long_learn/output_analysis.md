## 3. Архитектура модели

### FatigueCNN_Attention (основная модель)
- **249 763 параметра** (в 3 раза меньше, чем baseline LSTM — 739 041)
- IMU Encoder: Conv1D(6→64→128→256) + Temporal Attention → 256-dim
- Physio Encoder: Conv1D(4→32→64→128) + Temporal Attention → 128-dim
- Classifier: Linear(384→128→1)

### FatigueCNN_LSTM (baseline)
- 739 041 параметров
- BiLSTM поверх конкатенации с **seq_len=1** — что эквивалентно двум FC-слоям (LSTM не вносит реальной временной динамики)

```
Базовая модель:
  Средняя степень: 0.7595 → Выраженное
  Максимальная:    0.8738 → Выраженное
  Предсказано: 2/2 (100%) — все окна = усталость

  Максимальная:    0.8058 → Выраженное

```

### Интерпретация Fatigue Degree

- Интерпретация `sigmoid(logit)` как "степени переутомления" — это **маппинг вероятности бинарного классификатора на непрерывную шкалу**. Это **не калиброванная оценка** степени усталости.
- Уровни ("Лёгкое", "Умеренное", "Выраженное") — **произвольные эвристические пороги**, не подкреплённые клинической или физиологической валидацией.
- На 2 тестовых окнах нет возможности судить о качестве непрерывной оценки.

## 10. Сводка критических ошибок

### 🔴 Критические ошибки (влияют на валидность результатов)

1. **LOSO на 5 из 294 субъектов** — категорически недостаточная выборка для LOSO-валидации. Результат F1=0.83±0.17 статистически ненадёжен и завышен из-за микро-выборок WSD4FEDSRM.

2. **Персонализация на 6 окнах (2 в тесте)** — демонстрация персонализации бесполезна при таком объёме данных. ROC-AUC = nan. Результат F1=1.0 ничего не доказывает.

3. **BiLSTM с seq_len=1 в baseline** — BiLSTM применяется к единственному вектору (256+128=384), что не даёт никакого рекуррентного моделирования. Это ошибка проектирования baseline: сравнение с ним необъективно, т.к. baseline заведомо содержит архитектурный дефект.

4. **Fatigue Degree без калибровки** — sigmoid(logit) интерпретируется как "степень переутомления" (0–1), но нет:
   - Температурного скейлинга или Platt scaling
   - Сопоставления с RPE или физиологическими маркерами
   - Валидации шкалы на внешних данных

5. **Переобучение: лучшая модель на 3-й эпохе из 50** — модель моментально переобучается. Train loss падает, Val loss растёт. Это указывает на недостаточную регуляризацию или архитектурные проблемы.

### 🟡 Существенные замечания

6. **Несбалансированность доменов**: PhysioNet = 64%, WSD4FEDSRM = 1.6%. Метрики hold-out dominant по PhysioNet. Нет оценки per-domain.

7. **WSD4FEDSRM: 204 субъекта, 1124 окна** — в среднем 5.5 окон на субъекта. Эти субъекты раздувают число субъектов (294) и LOSO-фолдов, но не добавляют значимого объёма данных.

8. **Сравнение LSTM vs Attention не является fair**: LSTM обучается 30 эпох без аугментации, а Attention — 50 эпох с аугментацией. Для корректного сравнения нужно: (a) LSTM + аугментация, (b) Attention без аугментации.

9. **Аугментация не даёт измеримого улучшения**: F1-macro = 0.8028 (LSTM, no aug) = 0.8028 (Attention, aug). 9 техник аугментации не помогли улучшить обобщающую способность.

10. **Имя файла модели `best_model_lstm.pth`** для модели FatigueCNN_Attention — вводящее в заблуждение.

## 11. Решается ли задача определения степени переутомления?

### Бинарная классификация (норма vs усталость)

**Частично да.** Модель показывает F1=0.80 и AUC=0.89 на hold-out, что указывает на способность отличать классы лучше случайного. Однако:
- Единственный репрезентативный LOSO-субъект показал F1=0.64 — слабый результат
- Нет полноценной LOSO-оценки
- Переобучение на 3-й эпохе ставит под вопрос робастность

### Количественная оценка степени переутомления

**Нет.** Задача определения *степени* (а не бинарного наличия) переутомления **не решена**:
- Нет калибровки вероятностей
- Пороги "Лёгкое/Умеренное/Выраженное" — произвольные, без клинической валидации
- Шкала `sigmoid(logit)` не имеет физиологического смысла
- Для оценки степени нужны ординальные метки (RPE-шкала) или непрерывная разметка, а не бинарная

### Общий вывод

Ноутбук демонстрирует **работающий пайплайн** для бинарной классификации утомления из IMU/Physio данных с разумными (но не выдающимися) метриками. Однако **значительная часть выводов завышена или не подтверждена данными**:
- LOSO-оценка нерепрезентативна
- Персонализация не продемонстрирована
- Сравнение моделей не показывает преимущества Attention
- "Степень переутомления" — маркетинговая переформулировка вероятности бинарного классификатора

Для реального использования необходимы: полная LOSO-оценка (все 294 субъекта), калибровка вероятностей, per-domain метрики и валидация на независимом тестовом наборе.
