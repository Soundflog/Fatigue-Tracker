# Спецификация входных и выходных данных модели FatigueCNN_LSTM

## Модель: FatigueCNN_LSTM

Двухветвевая (dual-branch) гибридная архитектура CNN + BiLSTM для определения степени переутомления спортсмена.
- **Параметров:** ~474 тыс.
- **Задача:** бинарная классификация с выводом непрерывной оценки (степени переутомления)

---

## Входные данные

Модель принимает **три тензора** одновременно:

### 1. `x_imu` — данные инерциального датчика (IMU)

| Атрибут | Значение |
|---------|----------|
| Форма   | `(batch_size, 100, 6)` |
| Тип     | `torch.float32` |
| Диапазон | Z-нормализованные значения (стандартизованы по субъекту) |

**Каналы (ось 2):**

| Индекс | Канал | Описание | Единица |
|--------|-------|----------|---------|
| 0 | `ax` | Ускорение по оси X | м/с² (нормализовано) |
| 1 | `ay` | Ускорение по оси Y | м/с² (нормализовано) |
| 2 | `az` | Ускорение по оси Z | м/с² (нормализовано) |
| 3 | `gx` | Угловая скорость по X | рад/с (нормализовано) |
| 4 | `gy` | Угловая скорость по Y | рад/с (нормализовано) |
| 5 | `gz` | Угловая скорость по Z | рад/с (нормализовано) |

- **Окно:** 100 отсчётов (при 100 Hz ≈ 1 сек; при 256 Hz ≈ 0.39 сек после ресэмплинга)
- **Источники:** Zenodo IMU dataset, 4TU Marotta dataset, PhysioNet (канал ACC)

---

### 2. `x_physio` — физиологические сигналы

| Атрибут | Значение |
|---------|----------|
| Форма   | `(batch_size, 100, 4)` |
| Тип     | `torch.float32` |
| Диапазон | Z-нормализованные значения; **нули** для субъектов без физиологии |

**Каналы (ось 2):**

| Индекс | Канал | Описание | Частота источника |
|--------|-------|----------|-------------------|
| 0 | `bvp` | Blood Volume Pulse (фотоплетизмография / PPG) | 64 Hz (PhysioNet) / ~100 Hz (WSD4FEDSRM) |
| 1 | `eda` | Electrodermal Activity (кожно-гальваническая реакция) | 4 Hz |
| 2 | `temp` | Температура кожи | 4 Hz |
| 3 | `hr` | Heart Rate (ЧСС) | 1 Hz |

- **Источники:** PhysioNet Wearable Device Dataset (Empatica E4), WSD4FEDSRM (PPG → канал bvp)
- **Ресэмплинг:** все каналы ресэмплированы до 100 точек на окно через интерполяцию
- **Для субъектов без физиологии (Zenodo, 4TU):** тензор содержит нули — маска `has_physio=False` зануляет вклад этой ветки в слиянии
- **Частичная физиология (WSD4FEDSRM):** только канал `bvp` (индекс 0) заполнен; каналы `eda`, `temp`, `hr` равны нулю; маска `has_physio=True` (physio-ветка активирована)

---

### 3. `has_physio` — маска наличия физиологических данных

| Атрибут | Значение |
|---------|----------|
| Форма   | `(batch_size,)` |
| Тип     | `torch.bool` |

| Значение | Смысл |
|----------|-------|
| `True`  | Субъект из PhysioNet — physio-ветка активна (все 4 канала) |
| `True`  | Субъект из WSD4FEDSRM — physio-ветка активна (только `bvp`, остальные = 0) |
| `False` | Субъект из Zenodo или 4TU — physio-ветка заземлена нулями |

---

## Нормализация входных данных (предобработка)

Перед подачей в модель все сигналы должны быть нормализованы по методу **Z-score, рассчитанному на данных обучающей выборки**:

```python
# Для продакшн-инференса — использовать параметры, сохранённые при обучении
x_normalized = (x_raw - mean_train) / (std_train + 1e-8)
```

**Важно:** нормализацию надо считать **по каждому каналу отдельно** и **по всей обучающей выборке**, а не по отдельному окну.

## Выходные данные

### Логиты (сырой выход)

```python
logits = model(x_imu, x_physio, has_physio)
# Форма: (batch_size,), torch.float32
# Диапазон: (-∞, +∞)
```

### Степень переутомления (Fatigue Degree)

```python
fatigue_degree = torch.sigmoid(logits)
# Форма: (batch_size,), torch.float32
# Диапазон: [0.0, 1.0]
```

| Значение `fatigue_degree` | Интерпретация |
|--------------------------|---------------|
| 0.00 – 0.20 | Норма — переутомление отсутствует |
| 0.20 – 0.45 | Лёгкое утомление |
| 0.45 – **threshold** | Пограничное состояние |
| **threshold** – 0.75 | Умеренное переутомление |
| 0.75 – 1.00 | Выраженное переутомление |

> **threshold** (по умолчанию ≈ 0.5) определяется автоматически при обучении методом поиска максимального F1-macro на валидационной выборке (`find_best_threshold`).

### Бинарная метка

```python
y_pred = (fatigue_degree >= threshold).long()
# 0 = норма, 1 = переутомление
```

## Персонализация (Subject-specific Fine-tuning)

Для адаптации модели под конкретного пользователя требуется минимальный набор размеченных окон:

| Параметр | Рекомендуемое значение |
|----------|------------------------|
| Минимум окон | 30–50 (15–50 сек данных) |
| Окон для fine-tuning | 100–300 |
| Этапов fine-tuning | 2 (заморозка энкодеров → разморозка верхних слоёв) |
| LR Stage 1 | 0.001 (только классификатор) |
| LR Stage 2 | 0.0001 (LSTM + conv3 + классификатор) |

### Что сохраняет персонализация

```
base_model (pre-trained)
    ├── imu_encoder.conv1  ← заморожено в Stage 1, заморожено в Stage 2
    ├── imu_encoder.conv2  ← заморожено в Stage 1, заморожено в Stage 2
    ├── imu_encoder.conv3  ← заморожено в Stage 1, ОБУЧАЕТСЯ в Stage 2 (lr/10)
    ├── physio_encoder.conv3 ← аналогично
    ├── lstm               ← заморожено в Stage 1, ОБУЧАЕТСЯ в Stage 2 (lr/10)
    └── classifier         ← ОБУЧАЕТСЯ в обоих этапах
```

## Ограничения и допущения

1. **Частота дискретизации:** входные данные должны быть ресэмплированы до одной частоты и нарезаны на 100-точечные окна до подачи в модель.
2. **Нормализация:** Z-score параметры (mean, std) должны быть получены с обучающей выборки и применены к новым данным.
3. **Гироскоп:** если доступен только акселерометр (без гироскопа), каналы `gx, gy, gz` заполняются нулями.
4. **Physio без IMU:** текущая архитектура требует IMU-ветку как основную; physio-ветка опциональна.
5. **Степень переутомления:** значение `fatigue_degree` — вероятностная оценка, не клинический показатель.
6. **Частичная физиология:** если доступен только один physio-сигнал (например, PPG из WSD4FEDSRM), он помещается в канал `bvp` (индекс 0), остальные каналы (`eda`, `temp`, `hr`) заполняются нулями, а `has_physio=True`. Модель получает активированную physio-ветку с одним значимым каналом — это допустимо, поскольку нормализация и механизм внимания адаптируются к нулевым каналам.
