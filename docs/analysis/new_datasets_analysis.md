# Анализ новых датасетов-кандидатов для расширения AFC-проекта

Дата анализа: 2026-03-03.

---

## Контекст и критерии отбора

Задача проекта — **определение степени физического переутомления человека/спортсмена** по данным носимых устройств (IMU-акселерометр и гироскоп, PPG/фотоплетизмография, EDA, HR).
Целевой конечный пользователь — обычный потребитель с фитнес-браслетом/смарт-часами, поэтому приоритет отдан датчикам, доступным на массовом рынке (acc, gyro, PPG/HR).

Критерии оценки датасета для включения в проект:
1. Прямая или косвенная **разметка физического переутомления** (не когнитивного).
2. Наличие **IMU или физиологических сигналов** с носимых датчиков.
3. Официальная публикация: **лицензия + DOI** (желательно связанная научная статья).
4. Техническая совместимость с текущей **двухветвевой архитектурой CNN+LSTM** (IMU-ветка / Physio-ветка).

Текущая база (AFC v3): Zenodo 7997851, 4TU 14307743, PhysioNet 1.0.1 — 69 субъектов, бег + велотренировки/спринты, IMU + BVP/EDA/TEMP/HR.

## Датасет 1 — WSD4FEDSRM (Zenodo 8415066)

**Полное название:** Wearable sensor data for fatigue estimation during shoulder rotation movements
**DOI:** 10.5281/zenodo.8415066
**Лицензия:** CC BY 4.0
**Опубликован:** October 2023
**Научная статья:** финансирование Science Foundation Ireland (INSIGHT, CONNECT, Confirm); датасет цитируется, статья предположительно опубликована в профильном журнале (IEEE/Sensors).

### Описание

34 участника выполняли **плечевые вращения** (внутренняя и внешняя ротация плеча, диапазон 30–40°) до физического утомления.

**Сенсоры:**

| Датчик | Размещение | Каналы |
|--------|-----------|--------|
| IMU (acc, gyro, mag) | Предплечье, Кисть, Таз, Плечо, Грудина, Верхняя рука | acc_x/y/z, gyr_x/y/z, mag_x/y/z (на каждом сегменте) |
| EMG | 6 мышц плечевого пояса | anterior_deltoid, infraspinatus, lat_dorsi, pect_major, post_deltoid, upper_trapezius |
| PPG | Запястье | bvp/ppg |
| Borg RPE | Субъективная оценка нагрузки | self-report |
| Karolinska Sleepiness Scale | Уровень бодрости | self-report |

**Размер:** 392.7 MB (zip), 34 субъекта × несколько упражнений.

### На сколько подходит к задаче?

**Оценка: ВЫСОКАЯ** — датасет создан _именно_ для задачи fatigue estimation с носимых устройств.

Что ценно:
- Прямая разметка физического переутомления по протоколу нагрузки + Borg RPE.
- IMU + PPG — совпадает с целевым набором сенсоров потребительских устройств.
- Расширяет корпус за счёт **силовых/изолированных упражнений** (вращения плеча): иной паттерн движения, чем бег → улучшает обобщаемость модели.
- 34 субъекта — хороший масштаб.
- Многопозиционный IMU (6 точек) → можно выбрать оптимальный сегмент.

Что использовать:
- **IMU → IMU-ветка**: `acc_forearm` и `gyr_forearm` (или `shoulder`/`sternum`) → каналы ax, ay, az, gx, gy, gz. Магнитометр (`mag_*`) отбросить — недоступен на большинстве потребительских устройств.
- **PPG → Physio-ветка**: `ppg.csv` → канал `bvp`. Остальные physio-каналы (EDA, TEMP, HR) в этом датасете отсутствуют → заполнить нулями с `has_physio=True` только для BVP.
- **EMG → не использовать**: EMG не доступен на потребительских носимых.

### Влияние на существующие датасеты

Существующая структура NPZ не меняется. Нужно:
1. Добавить `io_wsd4fedsrm.py` в `afc/` (по аналогии с `io_4tu.py`).
2. Выбрать один сегмент IMU (рекомендуется `sternum` → ближайший аналог поясничного IMU из Zenodo/4TU) или `shoulder`.
3. Разметка: шаги ранней фазы (первые N повторений) → label=0, шаги финальной фазы (до отказа / RPE>7) → label=1. Borg RPE даёт точный вектор разметки.
4. Гармонизация частоты (неизвестна точно → вероятно ~100–200 Hz) → ресемплинг до 100 точек/окно.

### Сложность интеграции

**Умеренная.** Структура (CSV по субъектам, разделение по папкам) аналогична PhysioNet. Основные работы:
- Написать загрузчик.
- Определить частоту дискретизации (из description.txt в архиве).
- Построить схему разметки по Borg RPE или по номеру повторения.
- Выбрать один-два сегмента IMU для унификации.

## Датасет 2 — STS Fatigue Data (Figshare 15001362)

**Полное название:** STS fatigue data (Sit-to-Stand exercise, physical fatigue)
**DOI:** 10.6084/m9.figshare.15001362 (Figshare)
**Лицензия:** предположительно CC BY 4.0 (стандарт Figshare) — **уточнить при загрузке**
**Упражнение:** Sit-to-Stand (STS) — вставание со стула

> **Примечание:** JavaScript-защита Figshare заблокировала автоматическое чтение страницы. Информация основана на описании пользователя и общем профиле Figshare-датасетов. Перед включением необходимо вручную проверить страницу и наличие связанной публикации.

### Описание

Данные движения при выполнении упражнения STS (садиться–вставать) с нарастающей усталостью. STS — стандартный клинический тест функциональных возможностей нижних конечностей (особенно у пожилых/реабилитационных пациентов).

**Предполагаемая структура:**
- IMU-данные (акселерометр ± гироскоп), вероятно с таза/ноги/запястья.
- Разметка: ранние vs поздние повторения, или до/после протокола усталости.

### На сколько подходит к задаче?

**Оценка: НИЗКАЯ / ПОГРАНИЧНАЯ.**

Аргументы за:
- Прямое физическое утомление.
- Вероятно IMU-сигналы → совместимо с IMU-веткой.
- Функциональное упражнение (актуально для пожилых, реабилитация, или как базовое тестовое движение).

Аргументы против:
- **Только одно упражнение** с очень специфичным паттерном — ограниченная обобщаемость на спортивные задачи.
- STS — движение из клинической/реабилитационной области, а не спортивной. Контингент участников, скорее всего, не "спортсмены".
- Нет информации о PPG/физиологических сигналах.
- **Научная статья не верифицирована** при автоматическом анализе — риск, что датасет не рецензирован.
- Небольшой объём (предположительно).

**Рекомендация:** датасет можно рассмотреть только как _дополнение для transfer learning_ (предобучение на функциональных движениях нижних конечностей) или как тест обобщаемости на нетипичное движение. В основной обучающий стек не включать без ручной проверки наличия статьи и описания протокола.

### Влияние на существующие датасеты

Минимальное при использовании только как тестового набора. При включении в обучение — нужен новый загрузчик и схема разметки.

### Сложность интеграции

**Высокая (неопределённость структуры)** — структура данных неизвестна до ручного скачивания. Сначала необходимо:
1. Открыть страницу Figshare вручную, скачать архив.
2. Проверить формат (CSV/MAT/другой), частоту, каналы.
3. Найти связанную публикацию (обязательное условие использования).

## Датасет 3 — Wearable IMU Dataset for HAR (Zenodo 15830858)

**Полное название:** Wearable IMU Sensor Dataset for Human Activity Recognition (HAR)
**DOI:** 10.5281/zenodo.15830858 (v2), последняя версия: 10.5281/zenodo.15857137 (v3, July 2025)
**Лицензия:** CC BY 4.0
**Опубликован:** June–July 2025
**Научная статья:** **отсутствует** (0 цитирований, очень свежая публикация без связанной статьи на момент анализа)

### Описание

Датасет IMU-сигналов для **распознавания активности (HAR)**, а не для определения переутомления.

**Сенсоры:**
- 5 IMU (50 Hz): Chest, Left Hand, Right Hand, Left Knee, Right Knee
- Каналы на каждом: quaternion (q_w, q_x, q_y, q_z) + acc (a_x, a_y, a_z) + gyro (g_x, g_y, g_z)

**Метки:** классы активностей (вид движения/упражнения) — **не метки переутомления**.

### На сколько подходит к задаче?

**Оценка: НЕ ПОДХОДИТ для прямого использования.**

Критические проблемы:
1. **Нет разметки переутомления.** Датасет размечен по типам активностей (HAR), не по степени усталости. Чтобы использовать для fatigue detection, нужно было бы знать, через сколько повторений/минут наступает переутомление — эта информация в датасете отсутствует.
2. **Нет научной статьи.** Датасет опубликован в июне–июле 2025 и не имеет ни одного цитирования на дату анализа (март 2026). Это критический пробел для академически обоснованного включения.
3. **Потенциально полезен только для pretraining** IMU-энкодера (self-supervised representation learning) — модель учится распознавать движения, затем fine-tune на fatigue. Но для этого достаточно уже имеющихся датасетов.

Что могло бы быть полезным:
- Разнообразие активностей (грудь, колени, руки) — расширяет понимание IMU-паттернов разных движений.
- Хорошее техническое оформление (README, Jupyter notebook с кодом загрузки).

**Рекомендация:** не включать в текущий проект. Вернуться к переоценке, если датасет получит scientific backing (статья) и появятся версии с fatigue-разметкой.

### Влияние на существующие датасеты

Не применимо — датасет не рекомендован к включению.

## Уходим ли мы от тематики?

### Короткий ответ: нет, если соблюдать границы.

**Исходная тематика** (как вы её сформулировали):
- Определение **физического** переутомления.
- Интеграция с **носимыми устройствами** (потребительский уровень: acc, gyro, PPG/HR).
- Широкий спектр активностей: бег, приседания, лёгкая атлетика, силовые упражнения.

**Где граница:**
- ✅ WSD4FEDSRM — плечевые вращения с IMU+PPG — это силовое изолированное упражнение с носимых датчиков. Полностью в теме.
- ⚠️ STS — функциональное клиническое упражнение. Находится на грани "спорта" и реабилитации. Допустимо расширение в сторону фитнеса для обычного потребителя (пожилые люди, ЛФК), но основной фокус — всё же спортивные нагрузки.
- ❌ HAR (Zenodo 15830858) — датасет для распознавания активностей, нет fatigue. Это уже другая задача (что делает человек, а не насколько он устал).
- ❌ FatigueSet (ранее исключён) — ментальная усталость. За пределами.

**Рекомендуемая стратегия расширения**, чтобы оставаться в тематике:

```
核心 (бег, велотренировки)            РАСШИРЕНИЕ (другие физические нагрузки)
Zenodo 7997851 + 4TU + PhysioNet  ────────►  WSD4FEDSRM (плечо)
                                             SWIFTIES (верхняя конечность)
                                             STS (после верификации)

ЗАПРЕЩЁННЫЕ ЗОНЫ:
- Когнитивная усталость (FatigueSet)
- HAR без fatigue-разметки
- EEG / фМРТ / клинические датчики без IMU/PPG аналогов
```

| Приоритет | Датасет | Роль | Активность |
|-----------|---------|------|------------|

| 2 (текущий) | 4TU 14307743 | Cross-dataset validation (бег) | Бег |

**Итогом расширения** (после WSD4FEDSRM) станет покрытие задачи определения переутомления для:
- Кардиовыносливостных нагрузок (бег, велосипед, спринт).
- Изолированных/силовых движений верхней конечности (плечо, рука).
- Функциональных движений (STS — условно).

Это соответствует исходной цели: потребительское носимое устройство (фитнес-браслет с acc+gyro и опционально PPG) применяется к широкому спектру физических нагрузок без специализированного лабораторного оборудования.
