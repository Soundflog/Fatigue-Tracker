## 1. Введение и постановка задачи

### 1.1 Цель проекта

Разработка системы автоматического определения переутомления спортсмена на основе данных носимых устройств (фитнес-трекеры, смарт-часы, фитнес-браслеты) с использованием моделей машинного обучения и нейронных сетей.

### 1.2 Требования к системе

| Требование | Описание |
|------------|----------|
| **Источники данных** | Носимые устройства: акселерометр (IMU), гироскоп, PPG (фотоплетизмография), EDA (электродермальная активность), температура, частота сердечных сокращений (HR) |
| **Дисциплины** | Смешанные: бег, велотренировки (cycling), анаэробные спринты, мультимодальные нагрузки |
| **Тип модели** | CNN (свёрточный энкодер, двухветочная архитектура FatigueWristNet) |
| **Персонализация** | Поддержка предобучения (pretrained) с дообучением под конкретного спортсмена |
| **Режим работы** | Оффлайн-анализ (с возможностью расширения до real-time) |

### 1.3 Формализация задачи

**Бинарная классификация временных рядов:**

Пусть $\mathbf{X} \in \mathbb{R}^{T \times C}$ — многоканальный временной ряд, где:
- $T$ — длина окна (количество отсчётов)
- $C$ — количество каналов датчиков

Требуется построить функцию $f_\theta: \mathbb{R}^{T \times C} \rightarrow [0, 1]$, такую что:

$$\hat{y} = f_\theta(\mathbf{X}) = P(\text{fatigue} | \mathbf{X})$$

где $\hat{y}$ — вероятность состояния переутомления.

### 2.2 Детализация датасетов

#### 2.2.1 Zenodo Running IMU Dataset

**Описание:**

**Структура данных:**
```

├── Accel_Z_all.csv    # label: F (fatigue) или NF (not fatigue)

├── Gyro_Y_all.csv
├── Gyro_Z_all.csv
├── Accel_mag_all.csv  # опционально: магнитуда ускорения
```

**Разметка:**

- 19 бегунов (из 8 файлов), MVN Link (Xsens) — 8 IMU на сегментах тела

**Структура данных:**
```

├── p001_HDSL_postfatigue1200m.mat  # забег после утомления
├── p001_strides.mat             # сегментированные шаги
```

**Описание:**

**Выбор сегмента IMU:**
Для гармонизации с Zenodo/4TU используется сегмент **Sternum** (грудина) как ближайший аналог поясничного/грудинного IMU из других датасетов. Возможный альтернативный вариант: `Shoulder`.

**Разметка (по Borg RPE, каждые 10 сек):**
- RPE ≤ 11 → label = 0 (норма, лёгкая нагрузка)
- RPE ≥ 14 → label = 1 (усталость, тяжёлая нагрузка)
- RPE 12–13 → исключаются как пограничные (переходная зона)
- Borg RPE = 20 → конец упражнения (максимальное усилие)

**Стратегия PID (идентификатор субъекта):**

> **ВАЖНО (v4.0):** Каждый субъект выполняет 6 задач (3 уровня нагрузки × 2 типа ротации). Ранее PID назначался по формуле `wsd_{subject}_{task}` (например, `wsd_1_task1_35i`), что приводило к **инфляции PID: 34 человека × 6 задач = 204 PID**, при этом на каждый PID приходилось лишь **~5.5 окон**. Это делало LOSO-валидацию бессмысленной (3–9 окон на фолд).
>
> **Исправление:** PID назначается **по человеку**: `wsd_1`, `wsd_2`, ..., `wsd_34`. Все 6 задач одного человека объединяются. Результат: **34 PID, ~33 окна/PID** — достаточно для корректной LOSO-валидации.

**Ограничения:**
- Магнитометр (`mag_*`) не используется — ни один потребительский IMU его не предоставляет
- EMG не используется — недоступен на носимых устройствах
- Упражнение изолированное (только плечо) → domain shift относительно беговых датасетов
- Малое число окон на субъекта (~33) по сравнению с другими доменами (~316–2358)

### 2.4 Стратегия композиции

> **Изменение v6.0:** Фокус смещён на **запястные датчики** (браслет/часы). Датасеты Zenodo и 4TU (IMU на поясе/ногах) **исключены** из текущей версии — они не содержат данных запястья. PhysioNet (Empatica E4 на запястье) и WSD4FEDSRM (IMU на предплечье/кисти + PPG) были основными источниками в v6.0.

> **Изменение v7.0 (ablation study):** WSD4FEDSRM **исключён** из обучающей выборки по результатам сравнительного эксперимента (см. раздел 3). Доказано, что добавление WSD4FEDSRM ухудшает качество модели или не даёт преимуществ при существенном усложнении конвейера. **v7.0 использует исключительно PhysioNet.**

В v7.0 датасет состоит из одного источника — **PhysioNet** (Empatica E4 на запястье), два типа входов:

- **IMU-ветка** (PhysioNet ACC запястья, 3 канала + 3 нулевых канала gyro): инерциальные данные
- **Physio-ветка** (PhysioNet BVP/EDA/TEMP/HR): физиологические сигналы, присутствуют у всех субъектов (`has_physio = True` для всех окон)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                       ДАТАСЕТ (AFC v7.0)                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                           PhysioNet                                          │
│          (до 31 субъ., cycling/спринты, Empatica E4 на запястье)             │
│                        4-64 Hz, непрерывные записи                           │
├─────────────────────────────────────────────────────────────────────────────┤
│  IMU-каналы: ax, ay, az (gyro→0, нет гироскопа на E4)                      │
│  Physio-каналы: BVP, EDA, TEMP, HR (все 4/4 присутствуют)                  │
│  Целевое окно: 100 точек, stride=5s (без перекрытия)                        │
│  Разметка: 0 = pre-fatigue, 1 = post-fatigue                                │
│  Протоколы: AEROBIC + ANAEROBIC                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

> **Примечание:** Zenodo, 4TU и WSD4FEDSRM остаются доступны в репозитории как архивные данные и возможные источники для будущих расширений, но **не используются в v7.0**.

## 3. Ablation Study: обоснование выбора датасета

### 3.1 Постановка эксперимента

В рамках перехода к v7.0 проведён контролируемый ablation study для ответа на вопрос: **какой датасет (или их комбинация) даёт наилучшее качество FatigueWristNet?**

Три конфигурации с единым CONFIG:

| # | Название | Данные | Sub-windowing | N окон | N субъ. |
|---|---------|--------|---------------|--------|---------|
| 1 | `physionet_only` | PhysioNet (AEROBIC + ANAEROBIC) | Нет | 23 034 | 31 |
| 2 | `wsd4fedsrm_forearm` | WSD4FEDSRM Forearm | Да (×3) | 1 123 | 34 |
| 3 | `combined_wrist` | PhysioNet + WSD4FEDSRM Forearm | Да (WSD only) | 24 157 | 65 |

```python
def load_4tu_strides(data_dir: Path, segment: str = 'pelvis') -> dict:
    """
    Загрузка шагов из MAT файлов 4TU.
    
    Args:
        segment: сегмент тела ('pelvis', 'sternum', 'lfoot', 'rfoot')
    
    Returns:
        dict аналогичный load_zenodo_strides
    """
    all_acc, all_angvel, all_labels, all_pids = [], [], [], []
    
    for mat_file in sorted(data_dir.glob('p*_strides.mat')):
        pid = int(mat_file.stem.split('_')[0][1:])  # p001 -> 1
        
        mat = scipy.io.loadmat(mat_file)
        strides = mat['strides'][0, 0]
        
        # Извлекаем данные выбранного сегмента
        acc = strides[f'{segment}_nacc']      # (150, N_strides)
        angvel = strides[f'{segment}_angvel'] # (150, N_strides)
        
        n_strides = acc.shape[1]
        
        # Разметка: первая половина = 0, вторая половина = 1
        # (упрощённая эвристика, можно уточнить по файлам *_0-2k vs *_postfatigue)
        labels = np.zeros(n_strides, dtype=int)
        labels[n_strides // 2:] = 1
        
        all_acc.append(acc.T)           # -> (N, 150)
        all_angvel.append(angvel.T)
        all_labels.append(labels)
        all_pids.append(np.full(n_strides, pid))
    
    acc = np.vstack(all_acc)           # (total_strides, 150)
    angvel = np.vstack(all_angvel)
    
    return {
        'channels': {
            'ax': acc[:, :, 0] if acc.ndim == 3 else acc,
            'ay': acc[:, :, 1] if acc.ndim == 3 else np.zeros_like(acc),
            'az': acc[:, :, 2] if acc.ndim == 3 else np.zeros_like(acc),
            'gx': angvel[:, :, 0] if angvel.ndim == 3 else angvel,
            'gy': angvel[:, :, 1] if angvel.ndim == 3 else np.zeros_like(angvel),
            'gz': angvel[:, :, 2] if angvel.ndim == 3 else np.zeros_like(angvel),
        },
        'pids': np.concatenate(all_pids),
        'labels': np.concatenate(all_labels),
        'n_strides': len(np.concatenate(all_labels)),
        'stride_len': 150,
        'fs_original': 240,
    }
```

def load_physionet(data_dir: Path, 
                   protocols: list = ['AEROBIC', 'ANAEROBIC'],
                   window_sec: float = 5.0,
                   stride_sec: float = 2.5,
                   target_len: int = 100) -> dict:
    """
    Загрузка и нарезка окон из PhysioNet Wearable Dataset.
    
    Формат CSV (Empatica E4):
    - Строка 1: стартовое время (UTC)
    - Строка 2: частота (Hz)
    - Строки 3+: данные
    
    Args:
        data_dir: путь к Wearable_Dataset/
        protocols: список протоколов ['AEROBIC', 'ANAEROBIC']
        window_sec: длина окна в секундах
        stride_sec: шаг скользящего окна в секундах
        target_len: целевая длина после ресемплинга
    
    Returns:
        dict с X_imu, X_physio, y, pids, domain
    """
    all_X_imu = []
    all_X_physio = []
    all_labels = []
    all_pids = []
    
    for protocol in protocols:
        proto_dir = data_dir / protocol
        for subj_dir in sorted(proto_dir.iterdir()):
            pid = subj_dir.name  # 'S01', 'f03', etc.
            
            # Загрузка модальностей
            acc, acc_fs = load_empatica_csv(subj_dir / 'ACC.csv', n_cols=3)
            bvp, bvp_fs = load_empatica_csv(subj_dir / 'BVP.csv', n_cols=1)
            eda, eda_fs = load_empatica_csv(subj_dir / 'EDA.csv', n_cols=1)
            temp, temp_fs = load_empatica_csv(subj_dir / 'TEMP.csv', n_cols=1)
            hr, hr_fs = load_empatica_csv(subj_dir / 'HR.csv', n_cols=1)
            tags = load_tags(subj_dir / 'tags.csv')
            
            # Определяем общую длительность (по ACC, 32 Hz)
            total_sec = len(acc) / acc_fs
            n_windows = int((total_sec - window_sec) / stride_sec) + 1
            
            # Разметка фаз по tags
            labels = assign_fatigue_labels(tags, n_windows, 
                                           window_sec, stride_sec,
                                           protocol=protocol)
            
            # Нарезка окон для каждой модальности
            for w in range(n_windows):
                t_start = w * stride_sec
                t_end = t_start + window_sec
                
                # IMU: ACC (32 Hz) → 3 канала acc + 3 нуля (нет gyro)
                acc_win = extract_window(acc, acc_fs, t_start, t_end)
                imu_win = np.zeros((target_len, 6), dtype=np.float32)
                for ax in range(3):
                    imu_win[:, ax] = resample_stride(
                        acc_win[:, ax], target_len)
                # gx, gy, gz = 0 (нет гироскопа на Empatica E4)
                
                # Physio: BVP + EDA + TEMP + HR → 4 канала
                physio_win = np.zeros((target_len, 4), dtype=np.float32)
                physio_win[:, 0] = resample_stride(
                    extract_window(bvp, bvp_fs, t_start, t_end).flatten(),
                    target_len)
                physio_win[:, 1] = resample_stride(
                    extract_window(eda, eda_fs, t_start, t_end).flatten(),
                    target_len)
                physio_win[:, 2] = resample_stride(
                    extract_window(temp, temp_fs, t_start, t_end).flatten(),
                    target_len)
                physio_win[:, 3] = resample_stride(
                    extract_window(hr, hr_fs, t_start, t_end).flatten(),
                    target_len)
                
                all_X_imu.append(imu_win)
                all_X_physio.append(physio_win)
                all_labels.append(labels[w])
                all_pids.append(f"physionet_{pid}_{protocol.lower()}")
    
    return {
        'X_imu': np.array(all_X_imu),
        'X_physio': np.array(all_X_physio),
        'y': np.array(all_labels, dtype=np.int8),
        'pids': np.array(all_pids),
        'domain': np.array(['physionet'] * len(all_labels)),
    }

```python
    seg_name = segment  # 'sternum', 'shoulder', 'forearm', ...
    ex_folder = '30-40_ external rotation' \
        if exercise == 'external' else '30-40_ internal rotation'
    
    imu_dir = data_dir / 'EMG, IMU, and PPG data' / ex_folder
    borg_df = pd.read_csv(data_dir / 'Borg data' / 'borg_data.csv')
    
    all_X_imu, all_X_physio, all_labels, all_pids = [], [], [], []
    
    for subj_dir in sorted(imu_dir.iterdir()):
        if not subj_dir.is_dir():
            continue
        pid = int(subj_dir.name.split()[-1])  # 'Subject 1' -> 1
        
        # Загрузка IMU (acc + gyro, без mag)
        acc = np.loadtxt(
            subj_dir / 'IMU data' / seg_name.capitalize() / f'acc_{seg_name}.csv',
            delimiter=',')
        gyro = np.loadtxt(
            subj_dir / 'IMU data' / seg_name.capitalize() / f'gyr_{seg_name}.csv',
            delimiter=',')
        ppg = np.loadtxt(
            subj_dir / 'PPG data' / 'ppg.csv',
            delimiter=',')
        
        # Оценки RPE для данного субъекта и упражнения
        subj_borg = borg_df[
            (borg_df['subject'] == pid) & 
            (borg_df['exercise'] == exercise)
        ].sort_values('repetition')
        
        n_reps = len(subj_borg)
        rpe_scores = subj_borg['borg_rpe'].values
        
        # Сегментация сигнала по повторениям
        # Каждое повторение = цикл ддвижения; длина подбирается автоматически
        rep_len = len(acc) // n_reps  # приблизительная длина повторения
        
        for rep_idx in range(n_reps):
            rpe = rpe_scores[rep_idx]
            if 12 <= rpe <= 13:
                continue  # исключаем пограничные
            label = 0 if rpe <= 11 else 1
            
            s = rep_idx * rep_len
            e = s + rep_len
            
            # IMU: acc (3) + gyro (3) → 6 каналов
            imu_rep = np.hstack([
                acc[s:e, :3],
                gyro[s:e, :3]
            ])  # (rep_len, 6)
            X_imu = resample_stride(imu_rep, target_len)  # (100, 6)
            
            # Physio: только BVP (канал 0), EDA/TEMP/HR = 0
            X_physio = np.zeros((target_len, 4), dtype=np.float32)
            ppg_seg = ppg[s:e] if ppg.ndim == 1 else ppg[s:e, 0]
            X_physio[:, 0] = resample_stride(ppg_seg, target_len)
            
            all_X_imu.append(X_imu.astype(np.float32))
            all_X_physio.append(X_physio)
            all_labels.append(label)
            all_pids.append(pid)
    
    return {
        'X_imu': np.array(all_X_imu),
        'X_physio': np.array(all_X_physio),
        'y': np.array(all_labels, dtype=np.int8),
        'pids': np.array(all_pids),
        'domain': np.array(['wsd4fedsrm'] * len(all_labels)),
    }
```

> **Примечание:** `has_physio = True` для WSD4FEDSRM, но только канал `bvp` ([индекс 0]) заполнен. Каналы `eda`, `temp`, `hr` (индексы 1–3) — нули. Маска `has_physio` обозначает активацию physio-ветки в целом; вес отдельных каналов внутри ветки определяется нормализацией из данных.

### 4.5 Ресемплирование шагов

Приведение к единой длине методом линейной интерполяции:

```python
def resample_stride(stride: np.ndarray, target_len: int = 100) -> np.ndarray:
    """
    Ресемплирование одного шага до target_len точек.
    
    Args:
        stride: shape (original_len,) или (original_len, n_channels)
        target_len: целевое количество точек
    
    Returns:
        shape (target_len,) или (target_len, n_channels)
    """
    from scipy.interpolate import interp1d
    
    original_len = stride.shape[0]
    x_old = np.linspace(0, 1, original_len)
    x_new = np.linspace(0, 1, target_len)
    
    if stride.ndim == 1:
        f = interp1d(x_old, stride, kind='linear')
        return f(x_new)
    else:
        result = np.zeros((target_len, stride.shape[1]))
        for ch in range(stride.shape[1]):
            f = interp1d(x_old, stride[:, ch], kind='linear')
            result[:, ch] = f(x_new)
        return result
```

$$x_{new}[n] = \text{interp}\left(x_{old}, \frac{n}{L_{new}} \cdot L_{old}\right)$$

### 4.5 Нормализация

**Z-score по субъекту:**

$$\tilde{x}_{s,c} = \frac{x_{s,c} - \mu_{s,c}}{\sigma_{s,c} + \epsilon}$$

где $\mu_{s,c}$ и $\sigma_{s,c}$ — среднее и стандартное отклонение по всем шагам субъекта $s$ для канала $c$.

```python
def normalize_per_subject(X: np.ndarray, pids: np.ndarray) -> np.ndarray:
    """
    Z-score нормализация по субъектам.
    
    Args:
        X: shape (N_strides, stride_len, n_channels)
        pids: shape (N_strides,) — ID субъектов
    
    Returns:
        нормализованный X
    """
    X_norm = X.copy()
    for pid in np.unique(pids):
        mask = pids == pid
        subj_data = X[mask]  # (n_subj_strides, stride_len, n_channels)
        
        # Статистики по всем шагам субъекта
        mean = subj_data.mean(axis=(0, 1), keepdims=True)  # (1, 1, n_ch)
        std = subj_data.std(axis=(0, 1), keepdims=True) + 1e-8
        
        X_norm[mask] = (subj_data - mean) / std
    
    return X_norm
```

```python
    domains = np.concatenate([

        ['4tu'] * len(ftu['labels']),

        np.zeros(len(zenodo['labels']), dtype=bool),
        np.zeros(len(ftu['labels']), dtype=bool),

        np.ones(len(wsd['y']), dtype=bool),   # WSD: partial physio (BVP only)

    # Нормализация по субъектам (отдельно для IMU и physio)
    X_imu = normalize_per_subject(X_imu, pids)
    X_physio_all[has_physio] = normalize_per_subject(
        X_physio_all[has_physio], pids[has_physio])
    
    # Сохранение
    np.savez_compressed(
        output_path,
        X_imu=X_imu.astype(np.float32),
        X_physio=X_physio_all.astype(np.float32),
        y=y.astype(np.int8),
        pids=pids,
        domains=domains,
        has_physio=has_physio,
    )
```

## 5. Архитектура модели FatigueWristNet (CNN)

### 5.1 Выбор архитектуры

> **Изменение v7.0:** Архитектура упрощена до **CNN-only** (FatigueWristNet). LSTM-компонент исключён — единственный источник (PhysioNet) не требует моделирования межоконных зависимостей, а CNN-only показала лучшие результаты при меньшей сложности.

Для задачи классификации мультимодальных временных рядов с носимых датчиков выбрана **двухветочная CNN-архитектура FatigueWristNet**:

1. **IMUEncoderWithAttention** — CNN-энкодер с механизмом внимания для IMU-каналов (акселерометр)
2. **PhysioEncoder** — CNN-энкодер для физиологических каналов (BVP, EDA, TEMP, HR)
3. **Позднее слияние (late fusion)** — конкатенация эмбеддингов двух веток → классификатор

**Обоснование CNN-архитектуры:**
- CNN извлекает **инвариантные к сдвигу** паттерны из отдельных окон
- Attention-механизм позволяет модели фокусироваться на наиболее информативных временных участках
- Двухветочность решает проблему **модальной неоднородности** (IMU vs физиологические сигналы)
- Все данные из PhysioNet имеют physio-каналы (`has_physio = True` для всех окон), поэтому обе ветки активны всегда

### 5.4 Код модели (PyTorch)

> **Изменение v7.0:** Модель переименована в `FatigueWristNet`. LSTM-компонент удалён. IMU-энкодер дополнен Attention-механизмом.

```python
import torch
import torch.nn as nn


class ConvBlock1D(nn.Module):
    """Универсальный свёрточный блок."""
    def __init__(self, in_ch, out_ch, kernel, pool='max', dropout=0.2):
        super().__init__()
        self.block = nn.Sequential(
            nn.Conv1d(in_ch, out_ch, kernel, padding=kernel // 2),
            nn.BatchNorm1d(out_ch),
            nn.ReLU(inplace=True),
            nn.MaxPool1d(2) if pool == 'max' else nn.AdaptiveAvgPool1d(1),
            nn.Dropout(dropout) if pool == 'max' else nn.Identity(),
        )
    def forward(self, x):
        return self.block(x)


class IMUEncoderWithAttention(nn.Module):
    """CNN-энкодер с Attention для IMU каналов (acc + gyro)."""
    def __init__(self, in_channels=6, dropout=0.2):
        super().__init__()
        self.conv1 = ConvBlock1D(in_channels, 64, 7, 'max', dropout)
        self.conv2 = ConvBlock1D(64, 128, 5, 'max', dropout)
        self.conv3 = ConvBlock1D(128, 256, 3, 'max', dropout)
        self.attention = nn.Sequential(
            nn.Linear(256, 1),
            nn.Softmax(dim=1),
        )
        self.out_dim = 256
    
    def forward(self, x):
        x = x.transpose(1, 2)  # (B, T, C) -> (B, C, T)
        x = self.conv1(x)
        x = self.conv2(x)
        x = self.conv3(x)      # (B, 256, T')
        x = x.transpose(1, 2)  # (B, T', 256)
        attn = self.attention(x)  # (B, T', 1)
        x = (x * attn).sum(dim=1)  # (B, 256)
        return x


class PhysioEncoder(nn.Module):
    """CNN-энкодер для физиологических каналов (BVP, EDA, TEMP, HR)."""
    def __init__(self, in_channels=4, dropout=0.2):
        super().__init__()
        self.encoder = nn.Sequential(
            ConvBlock1D(in_channels, 32, 7, 'max', dropout),
            ConvBlock1D(32, 64, 5, 'max', dropout),
            ConvBlock1D(64, 128, 3, 'gap'),
        )
        self.out_dim = 128
    
    def forward(self, x):
        x = x.transpose(1, 2)  # (B, T, C) -> (B, C, T)
        x = self.encoder(x)
        return x.view(x.size(0), -1)  # (B, 128)


class FatigueWristNet(nn.Module):
    """
    CNN двухветочная модель для детекции утомления (v7.0).
    
    Архитектура:
    - IMU-ветка: IMUEncoderWithAttention (акселерометр, 6 каналов)
    - Physio-ветка: PhysioEncoder (BVP + EDA + TEMP + HR, 4 канала)
    - Late fusion: конкатенация → Classifier
    """
    def __init__(self, imu_channels=6, physio_channels=4, 
                 use_physio=True, dropout=0.2):
        super().__init__()
        
        self.use_physio = use_physio
        
        # Ветка 1: IMU encoder с Attention
        self.imu_encoder = IMUEncoderWithAttention(imu_channels, dropout)
        
        # Ветка 2: Physio encoder
        self.physio_encoder = PhysioEncoder(physio_channels, dropout) \
            if use_physio else None
        
        # Размер fusion-вектора
        fusion_dim = self.imu_encoder.out_dim  # 256
        if use_physio:
            fusion_dim += self.physio_encoder.out_dim  # + 128 = 384
        
        # Classifier head
        self.classifier = nn.Sequential(
            nn.Linear(fusion_dim, 128),
            nn.ReLU(inplace=True),
            nn.Dropout(0.5),
            nn.Linear(128, 1),
            nn.Sigmoid(),
        )
    
    def forward(self, x_imu, x_physio=None, has_physio=None):
        """
        Args:
            x_imu: (B, T, 6)
            x_physio: (B, T, 4), может быть None
            has_physio: (B,) bool mask
        
        Returns:
            (B,) — вероятность усталости
        """
        emb = self._encode_window(x_imu, x_physio, has_physio)
        return self.classifier(emb).squeeze(-1)
    
    def _encode_window(self, x_imu, x_physio, has_physio):
        """Кодирование одного окна через обе ветки."""
        imu_emb = self.imu_encoder(x_imu)  # (B, 256)
        
        if self.use_physio and x_physio is not None:
            physio_emb = self.physio_encoder(x_physio)  # (B, 128)
            if has_physio is not None:
                mask = has_physio.float().unsqueeze(1)  # (B, 1)
                physio_emb = physio_emb * mask
            return torch.cat([imu_emb, physio_emb], dim=1)  # (B, 384)
        
        return imu_emb  # (B, 256)
    
    def extract_features(self, x_imu, x_physio=None, has_physio=None):
        """Извлечение эмбеддингов для transfer learning."""
        return self._encode_window(x_imu, x_physio, has_physio)
```

## 6. Математическое описание модели

### 6.1 Операция одномерной свёртки

Для входного сигнала $\mathbf{x} \in \mathbb{R}^{T \times C_{in}}$ и ядра свёртки $\mathbf{W} \in \mathbb{R}^{K \times C_{in} \times C_{out}}$:

$$y_j[t] = \sum_{c=1}^{C_{in}} \sum_{k=0}^{K-1} W_{k,c,j} \cdot x_c[t + k - \lfloor K/2 \rfloor] + b_j$$

где:
- $K$ — размер ядра
- $C_{in}$ — число входных каналов
- $C_{out}$ — число выходных каналов (фильтров)
- $b_j$ — смещение j-го фильтра

### 6.2 Batch Normalization

Для мини-батча активаций $\{z_i\}_{i=1}^{B}$:

$$\hat{z}_i = \gamma \cdot \frac{z_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}} + \beta$$

где $\mu_B, \sigma_B^2$ — среднее и дисперсия по батчу, $\gamma, \beta$ — обучаемые параметры.

### 6.3 Функция активации ReLU

$$\text{ReLU}(z) = \max(0, z)$$

### 6.4 Max Pooling

$$y[t] = \max_{k=0}^{P-1} x[t \cdot S + k]$$

где $P$ — размер окна пулинга, $S$ — шаг (stride).

### 6.5 Global Average Pooling

$$\bar{y}_j = \frac{1}{T'} \sum_{t=1}^{T'} y_j[t]$$

### 6.6 Функция потерь

> **Изменение v7.0:** Используется **Focal Loss** вместо BCE для лучшей работы с дисбалансом классов.

$$\mathcal{L}_{focal} = -\frac{1}{N} \sum_{i=1}^{N} \alpha_t (1 - p_t)^\gamma \log(p_t)$$

где $p_t = \hat{y}_i$ если $y_i = 1$, иначе $p_t = 1 - \hat{y}_i$, а $\gamma = 2.0$ — фокусирующий параметр, снижающий вклад легко классифицируемых примеров.

### 7.2 Аугментация данных

> **Изменение v7.0:** Добавлен **SMOTE (Synthetic Minority Over-sampling Technique)** для балансировки классов на этапе предобработки (до обучения). SMOTE применяется только к тренировочной выборке для предотвращения утечки данных.

#### 7.2.0 SMOTE (балансировка классов)

SMOTE генерирует синтетические образцы миноритарного класса путём интерполяции между существующими образцами и их ближайшими соседями в пространстве признаков:

$$\mathbf{x}_{synth} = \mathbf{x}_i + \lambda \cdot (\mathbf{x}_{nn} - \mathbf{x}_i), \quad \lambda \sim \mathcal{U}(0, 1)$$

где $\mathbf{x}_i$ — образец миноритарного класса, $\mathbf{x}_{nn}$ — его случайный ближайший сосед (k-NN, k=5).

| Параметр | Значение по умолчанию |
|---|---|
| `sampling_strategy` | `'auto'` (= выравнивание классов) |
| `k_neighbors` | 5 |
| Применяется к | только train split |
| Реализация | `imblearn.over_sampling.SMOTE` |

```python
from imblearn.over_sampling import SMOTE

def apply_smote(X_imu_train, X_physio_train, y_train):
    """
    Применение SMOTE к тренировочной выборке.
    Для многомерных входов (N, T, C) -> flatten -> SMOTE -> reshape.
    """
    N = len(y_train)
    X_flat = np.hstack([
        X_imu_train.reshape(N, -1),
        X_physio_train.reshape(N, -1)
    ])
    
    smote = SMOTE(k_neighbors=5, random_state=42)
    X_resampled, y_resampled = smote.fit_resample(X_flat, y_train)
    
    imu_dim = X_imu_train.shape[1] * X_imu_train.shape[2]
    X_imu_res = X_resampled[:, :imu_dim].reshape(-1, *X_imu_train.shape[1:])
    X_physio_res = X_resampled[:, imu_dim:].reshape(-1, *X_physio_train.shape[1:])
    
    return X_imu_res, X_physio_res, y_resampled
```

> **Примечание:** SMOTE применяется **до** on-the-fly аугментаций (ниже), только на train-сплите, и используется совместно с WeightedRandomSampler.

Аугментация применяется **во время обучения** (on-the-fly) с вероятностью $p_{aug}$ для каждой техники независимо. Параметры управляются через секцию `augmentation` конфига YAML.

```python
def add_gaussian_noise(x: np.ndarray, sigma: float = 0.05) -> np.ndarray:
```

#### 7.2.2 Временно́е искажение (Time Warp)

Нелинейное растяжение/сжатие временной оси с помощью случайного сглаженного смещения:

$$x'[t] = x\bigl[t + \delta(t)\bigr], \quad \delta(t) \sim \text{smooth}\,\mathcal{N}(0, \sigma_t^2)$$

Кубическая интерполяция после деформации сохраняет размерность $(100, C)$.

| Параметр | Значение по умолчанию |
|---|---|
| $\sigma_t$ | 0.2 |
| knots | 4 |
| $p_{aug}$ | 0.4 |

```python
from scipy.interpolate import CubicSpline

def time_warp(x: np.ndarray, sigma: float = 0.2, knots: int = 4) -> np.ndarray:
    T, C = x.shape
    tt = np.linspace(0, T - 1, knots + 2)
    warp = np.concatenate([[0], np.random.normal(0, sigma * T, knots), [0]])
    warp_fn = CubicSpline(tt, warp)
    t_orig = np.arange(T)
    t_warped = np.clip(t_orig + warp_fn(t_orig), 0, T - 1)
    return np.stack([
        CubicSpline(t_orig, x[:, c])(t_warped) for c in range(C)
    ], axis=-1).astype(x.dtype)
```

#### 7.2.3 Dropout каналов

Случайное обнуление целого канала:

$$x'[:, c] = \mathbf{0}, \quad c \sim \mathcal{U}\{0, C-1\}~\text{с вероятностью}~p_{ch}$$

| Параметр | Значение по умолчанию |
|---|---|
| $p_{ch}$ | 0.15 |
| Применяется к | IMU и physio раздельно |

```python
def channel_dropout(x: np.ndarray, p: float = 0.15) -> np.ndarray:
    x = x.copy()
    for c in range(x.shape[1]):
        if np.random.rand() < p:
            x[:, c] = 0.0
    return x
```

#### 7.2.4 Масштабирование амплитуды (Magnitude Scaling)

$$x'[t] = s \cdot x[t], \quad s \sim \mathcal{U}(s_{min},\, s_{max})$$

Имитирует вариацию расположения датчика и разброс физических параметров субъектов.

| Параметр | Значение по умолчанию |
|---|---|
| $s_{min}$ | 0.8 |
| $s_{max}$ | 1.2 |
| $p_{aug}$ | 0.5 |
| Каналы | все |

```python
def magnitude_scale(x: np.ndarray,
                    s_min: float = 0.8, s_max: float = 1.2) -> np.ndarray:
    s = np.random.uniform(s_min, s_max, (1, x.shape[1])).astype(x.dtype)
    return x * s
```

#### 7.2.5 Случайная обрезка окна (Window Slicing)

Извлечение случайного подокна с последующим ресэмплингом до исходной длины:

$$x' = \text{resample}\bigl(x[s\!:\!s+L_{crop}],\, T\bigr), \quad s \sim \mathcal{U}(0, T - L_{crop})$$

| Параметр | Значение по умолчанию |
|---|---|
| $L_{crop}$ | $\lfloor 0.8 \cdot T \rfloor$ (≥ 80% окна) |
| $p_{aug}$ | 0.3 |

```python
from scipy.signal import resample

def window_slice(x: np.ndarray, crop_ratio: float = 0.8) -> np.ndarray:
    T, C = x.shape
    L = max(int(T * crop_ratio), 10)
    start = np.random.randint(0, T - L + 1)
    crop = x[start:start + L, :]
    return resample(crop, T, axis=0).astype(x.dtype)
```

#### 7.2.7 Вращение IMU (SO(3) Rotation Augmentation)

Применение случайной малоугловой матрицы вращения $R \in SO(3)$ к осям акселерометра и гироскопа:

$$\begin{pmatrix} a_x' \\ a_y' \\ a_z' \end{pmatrix} = R \begin{pmatrix} a_x \\ a_y \\ a_z \end{pmatrix}, \quad \theta \sim \mathcal{U}(-15°,\, 15°)$$

Матрицу вращения формируем через случайный единичный вектор оси $\hat{n}$ и угол $\theta$ по формуле Родрига:
$$R = I + \sin\theta\,[\hat{n}]_\times + (1 - \cos\theta)\,[\hat{n}]_\times^2$$

| Параметр | Значение по умолчанию |
|---|---|
| $\theta_{max}$ | 15° |
| $p_{aug}$ | 0.4 |
| Каналы | acc ([:, 0:3]) и gyro ([:, 3:6]) — отдельные матрицы |

```python
def imu_rotate(x_imu: np.ndarray, max_deg: float = 15.0) -> np.ndarray:
    """x_imu: (T, 6) — [ax, ay, az, gx, gy, gz]"""
    def rand_rot():
        theta = np.deg2rad(np.random.uniform(-max_deg, max_deg))
        n = np.random.randn(3); n /= np.linalg.norm(n)
        K = np.array([[0, -n[2], n[1]], [n[2], 0, -n[0]], [-n[1], n[0], 0]])
        return np.eye(3) + np.sin(theta) * K + (1 - np.cos(theta)) * K @ K

    x = x_imu.copy()
    x[:, :3] = x[:, :3] @ rand_rot().T   # акселерометр
    x[:, 3:6] = x[:, 3:6] @ rand_rot().T  # гироскоп
    return x.astype(x_imu.dtype)
```

#### 7.2.8 Mixup

Линейная интерполяция двух обучающих примеров и их меток:

$$x' = \lambda x_i + (1-\lambda) x_j, \quad y' = \lambda y_i + (1-\lambda) y_j, \quad \lambda \sim \text{Beta}(\alpha, \alpha)$$

| Параметр | Значение по умолчанию |
|---|---|
| $\alpha$ | 0.4 |
| $p_{aug}$ | 0.3 |
| Метки | soft labels (float, BCE-loss) |

```python
def mixup_batch(X_imu, X_physio, y, alpha: float = 0.4):
    lam = np.random.beta(alpha, alpha)
    idx = np.random.permutation(len(y))
    X_imu_mix = lam * X_imu + (1 - lam) * X_imu[idx]
    X_physio_mix = lam * X_physio + (1 - lam) * X_physio[idx]
    y_mix = lam * y + (1 - lam) * y[idx].astype(float)
    return X_imu_mix, X_physio_mix, y_mix
```

$$\eta_t = \eta_{min} + \frac{1}{2}(\eta_{max} - \eta_{min})\left(1 + \cos\left(\frac{t}{T_{max}}\pi\right)\right)$$

### 7.4 Алгоритм обучения

```python
def train_epoch(model, loader, optimizer, criterion, device):
    model.train()
    total_loss = 0
    for X, y in loader:
        X, y = X.to(device), y.to(device)
        
        optimizer.zero_grad()
        y_pred = model(X)
        loss = criterion(y_pred, y.float())
        loss.backward()
        
        # Gradient clipping
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
        
        optimizer.step()
        total_loss += loss.item()
    
    return total_loss / len(loader)
```

## 8. Протокол оценивания и метрики

### 8.1 Схемы валидации

#### 8.1.1 Leave-One-Subject-Out (LOSO)

Для $N$ субъектов выполняется $N$ итераций:
- Train: субъекты $\{1, ..., N\} \setminus \{i\}$
- Test: субъект $\{i\}$

**Агрегация:** среднее ± std по всем фолдам.

#### 8.1.2 Cross-Dataset Validation

```yaml
cross_dataset:
  train: ["zenodo", "4tu"]
  test: ["fatigueset"]
```

Проверка переносимости модели между датасетами.

### 8.2 Метрики качества

| Метрика | Формула | Интерпретация |
|---------|---------|---------------|
| **F1-macro** | $\frac{1}{2}(F1_0 + F1_1)$ | Балансированная точность по классам |
| **Balanced Accuracy** | $\frac{1}{2}\left(\frac{TP}{P} + \frac{TN}{N}\right)$ | Среднее между sensitivity и specificity |
| **ROC-AUC** | $\int_0^1 TPR(FPR^{-1}(t)) \, dt$ | Площадь под ROC-кривой |
| **PR-AUC** | $\int_0^1 Precision(Recall^{-1}(t)) \, dt$ | Важна при дисбалансе классов |
| **Brier Score** | $\frac{1}{N}\sum_i (y_i - \hat{y}_i)^2$ | Калибровка вероятностей |

### 9.3 Код персонализации

```python
def personalize_model(base_model, subject_data, epochs=20):
    """
    Дообучение модели под конкретного спортсмена.
    
    Args:
        base_model: предобученная модель
        subject_data: данные целевого субъекта (X, y)
        epochs: число эпох дообучения
    """
    model = copy.deepcopy(base_model)
    
    # Этап 1: заморозить backbone
    for param in model.conv1.parameters():
        param.requires_grad = False
    for param in model.conv2.parameters():
        param.requires_grad = False
    for param in model.conv3.parameters():
        param.requires_grad = False
    
    # Обучить только classifier
    optimizer = torch.optim.Adam(
        model.classifier.parameters(), 
        lr=0.001
    )
    
    # Fine-tuning loop
    for epoch in range(epochs // 2):
        train_epoch(model, subject_loader, optimizer, criterion)
    
    # Этап 2: разморозить conv3
    for param in model.conv3.parameters():
        param.requires_grad = True
    
    optimizer = torch.optim.Adam(
        list(model.conv3.parameters()) + 
        list(model.classifier.parameters()),
        lr=0.0001
    )
    
    for epoch in range(epochs // 2):
        train_epoch(model, subject_loader, optimizer, criterion)
    
    return model
```
