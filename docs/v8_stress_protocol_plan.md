# План v8.0: Интеграция STRESS-протокола PhysioNet

**Цель:** увеличить размер выборки за счёт STRESS-протокола из того же датасета PhysioNet (Empatica E4), не меняя архитектуру модели.

---

## 1. Текущее состояние (v7.0)

| Параметр | Значение |
|----------|----------|
| Протоколы | AEROBIC + ANAEROBIC |
| Субъектов | ~31 уникальных |
| Окон | ~4000–5000 (5 сек, stride=5 сек) |
| Сенсор | Empatica E4 (запястье) |
| Каналы | IMU (ACC 3 оси → 6 слотов) + Physio (BVP, EDA, TEMP, HR) |
| Метрики | Hold-out F1=0.80, LOSO F1=0.82 |

---

## 2. Что даёт STRESS-протокол

### 2.1 Доступные данные

| Параметр | Значение |
|----------|----------|
| Папок в STRESS/ | 37 (S01–S18, f01–f18, f14_a/f14_b) |
| Уникальных субъектов | ~36 (f14 — один человек в двух файлах) |
| Сенсор | **Empatica E4** — тот же самый |
| Каналы | ACC (32 Hz), BVP (64 Hz), EDA (4 Hz), TEMP (4 Hz), HR (1 Hz) |
| Формат файлов | Идентичен AEROBIC/ANAEROBIC |
| Разметка | Stress_Level_v1.csv / Stress_Level_v2.csv (самооценка 1–10 по фазам) |

### 2.2 Протокол STRESS

**V1 (S01–S18), 8 фаз:**
Baseline → Stroop → First Rest → TMCT → Second Rest → Real Opinion → Opposite Opinion → Subtract

**V2 (f01–f18), 7 фаз:**
Baseline → TMCT → First Rest → Real Opinion → Opposite Opinion → Second Rest → Subtract

Каждая фаза размечена в `tags.csv` (UTC-метки границ фаз) + самооценка стресса (1–10) в `Stress_Level_v*.csv`.

### 2.3 Перекрытие субъектов

Почти все субъекты участвовали во всех 3 протоколах:

| Протокол | Субъектов | Группа |
|----------|-----------|--------|
| STRESS | 37 (36 уникальных) | S01–S18, f01–f18 |
| AEROBIC | 31 (30 уникальных) | S01–S18 (без S12), f01–f13 |
| ANAEROBIC | 32 (31 уникальных) | S01–S18, f01–f13 |

**ВАЖНО:** одни и те же люди → при subject-split один субъект должен быть в одном сплите для ВСЕХ протоколов. Иначе — data leakage.

### 2.4 Ограничения качества данных (data_constraints.txt)

| Субъект | Проблема | Решение |
|---------|----------|---------|
| S02 | Дублирование сигналов после row ~49K (ACC) | **Обрезать** данные до границы дубликатов |
| f07 | Dock не снят — PPG и TEMP невалидны | **Использовать только ACC + EDA + HR** или исключить |
| f14 | BLE disconnect → f14_a (baseline) + f14_b (протокол) | **Склеить** f14_a + f14_b, PID = `physionet_f14` |

## 3. Обоснование интеграции

### 3.1 Физиологическое обоснование

Когнитивный стресс и физическая усталость вызывают **перекрывающуюся физиологическую реакцию**:

| Физиологический маркер | Физ. усталость | Когн. стресс | Канал E4 |
|------------------------|----------------|--------------|----------|
| ↑ ЧСС | ✅ | ✅ | HR |
| ↑ EDA (потоотделение) | ✅ | ✅ | EDA |
| ↑ Температура кожи | ✅ (↑) | ✅ (↓ периферия) | TEMP |
| Изменение BVP | ✅ | ✅ | BVP |
| Изменение движений | ✅ (↑ вариабельность) | ⚠️ (минимально) | ACC |

**Вывод:** Physio-ветка модели (BVP, EDA, TEMP, HR) получит полезный сигнал. IMU-ветка получит слабый сигнал (когнитивный стресс не вызывает явных двигательных паттернов), но модель уже обучена маскировать слабые ветки через `has_physio`.

### 3.2 Переформулировка задачи

| Версия | Задача | Данные |
|--------|--------|--------|
| v7.0 | Детекция **физической усталости** | AEROBIC + ANAEROBIC |
| v8.0 | Детекция **физиологической перегрузки** (включая когнитивную) | AEROBIC + ANAEROBIC + STRESS |

Это расширяет применимость модели: не только спорт, но и мониторинг рабочей нагрузки (пилоты, операторы, водители).

## 4. Стратегия разметки STRESS-данных

### 4.1 Вариант A: Phase-based (рекомендуемый)

Используем `tags.csv` для сегментации по фазам + самооценку стресса:

| Фаза | Тип | Метка |
|------|-----|-------|
| Baseline | Покой | **0** (normal) |
| First Rest | Покой | **0** (normal) |
| Second Rest | Покой | **0** (normal) |
| Stroop (V1) | Стресс | **1** (fatigue/stress) |
| TMCT | Стресс | **1** (fatigue/stress) |
| Real Opinion | Стресс | **1** (fatigue/stress) |
| Opposite Opinion | Стресс | **1** (fatigue/stress) |
| Subtract | Стресс | **1** (fatigue/stress) |

Дополнительный фильтр: **исключить окна, где самооценка стресса ≤ 2** (субъект не испытал стресса), чтобы не вносить шумные метки.

### 4.2 Вариант B: Stress-level thresholding

Бинаризация самооценки:
- Stress ≤ 3 → label **0**
- Stress ≥ 5 → label **1**
- Stress = 4 → **пропуск** (пограничная зона)

### 4.3 Рекомендация

**Вариант A** — проще в реализации, не требует интерполяции самооценки на уровне отдельных окон. Совместим с текущей функцией `assign_fatigue_labels_physionet()`.

## 7. Риски и митигация

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| **Шум в разметке STRESS** — сложно сопоставить теги с фазами | Средняя | Высокое | Вариант A (phase-based) + фильтр по самооценке стресса |
| **Доменный сдвиг** — STRESS-окна не похожи на физическую усталость | Высокая | Среднее | Ablation study (Фаза 3.2); при деградации — использовать только Physio-ветку |
| **IMU-шум в STRESS** — субъект сидит, минимальное движение | Высокая | Низкое | Маскировать IMU для STRESS (передать нули) или пометить отдельным флагом |
| **Subject leakage** — один субъект в train (AEROBIC) и test (STRESS) | Высокая | Критическое | **Subject-level split по base_id**: S01 во всех протоколах → один сплит |
| **Дисбаланс классов** сдвигается в сторону label=1 | Средняя | Среднее | SMOTE + WeightedSampler уже в пайплайне |
